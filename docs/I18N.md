# Wielojęzyczność (i18n)

Pierwszy kamień Fazy 1 z [ROADMAP.md](ROADMAP.md). Gra od teraz w PL / EN / DE,
kolejne języki dodaje się **dodając plik JSON** — bez dotykania kodu.

---

## Architektura — 3 pliki

```
www/
├── i18n.js              ← moduł: t(), setLanguage(), getLanguage(), ready
└── i18n/
    ├── pl.json          ← słownik PL
    ├── en.json          ← słownik EN
    └── de.json          ← słownik DE
```

### Jak gra wchodzi w interakcję

- **HTML**: każdy widoczny tekst ma atrybut `data-i18n="klucz"` (lub
  `data-i18n-html="..."` dla `<b>` w środku, lub `data-i18n-attr-title="..."`
  dla atrybutu `title`).
- **JS**: zamiast `"Tura: ..."` piszesz `i18n.t("status.turn", {who, color, goal})`.
- **Init**: skrypt gry uruchamia się **dopiero po `i18n.ready`**, żeby pierwsza
  klatka miała już prawidłowy język.

### Detekcja języka

1. `localStorage.pilkarzyki.lang` (jeśli użytkownik kiedyś wybrał)
2. `navigator.language[:2]` (pierwsze 2 znaki — `"pl"`, `"en-US"` → `"en"`)
3. Fallback: `pl`

Wybór z UI (przyciski PL/EN/DE w panelu) jest zapamiętywany w localStorage —
przeżywa restart apki.

---

## Dodawanie nowego języka (np. czeski `cs`)

1. Skopiuj `www/i18n/pl.json` → `www/i18n/cs.json`, przetłumacz wartości
2. W `www/i18n.js` linia 20:
   ```js
   const LANGS = ["pl", "en", "de", "cs"];
   ```
3. W `www/index.html` w sekcji `langSeg`:
   ```html
   <button data-lang="cs">CS</button>
   ```
4. W `www/sw.js` w `ASSETS` dorzuć `"./i18n/cs.json"` (cache offline)
5. Bump `CACHE` w `sw.js` (np. `pilkarzyki-v5`), żeby user dostał świeży cache

## Konwencje kluczy

```
<dział>.<element>            np. settings.modeBot
<dział>.<wariant><Cecha>     np. color.blueUpper / color.blueLower
                                 (różny case dla różnych pozycji w zdaniu)
```

**Zasady:**
- Klucze opisowe (`actions.newGame`), **nie** tekstowe (`nowa_gra`). Tekst może
  się zmienić, klucz zostaje.
- Tylko ASCII w kluczach. Polskie znaki tylko w wartościach.
- Interpolacja przez `{name}` (NIE `${name}`, NIE `%s`).
- HTML w wartościach OK, ale używaj `data-i18n-html` lub `<p data-i18n-html="...">`
  zamiast `data-i18n`.

## Plural (na przyszłość)

Polski ma 3 formy mnogie (`1 mecz / 2 mecze / 5 meczów`). Gdy dojdą statystyki,
rozszerzymy `t()` o `Intl.PluralRules`:

```json
"stats.played": {
  "one": "{count} mecz",
  "few": "{count} mecze",
  "many": "{count} meczów"
}
```

```js
i18n.t("stats.played", {count: 5})  // → "5 meczów"
```

Na razie nie potrzeba — wszystkie aktualne stringi są singularne.

## Daty i liczby

**Nie sklejaj liczb ręcznie.** Używaj `Intl.NumberFormat` / `Intl.DateTimeFormat`
z aktualnym językiem (`i18n.getLanguage()`). To darmowe, działa w przeglądarce.

---

## Gotchas

1. **Niemiecki jest długi.** „Ustawienia prywatności (zgoda na reklamy)"
   po niemiecku to „Datenschutz (Werbe-Zustimmung)" — krótsze, ale w wielu
   miejscach jest na odwrót. Przyciski elastyczne, żadnych sztywnych szerokości.
2. **Klucze które nie istnieją** zwracają sam klucz jako fallback. Zobaczysz
   `actions.newGame` zamiast napisu — wtedy wiesz że masz literówkę.
3. **Jeśli język w JSON nie ma klucza** który jest w PL — wpisz po angielsku
   albo zostaw jak w PL. Nigdy nie usuwaj klucza z JSON gdy istnieje w kodzie.

## Test po zmianach

```bash
node -e "
const fs=require('fs');
for(const lang of ['pl','en','de']){
  const d=JSON.parse(fs.readFileSync('www/i18n/'+lang+'.json'));
  console.log(lang, Object.keys(d).length, 'kluczy');
}
"
```

Wszystkie liczby kluczy powinny być **identyczne**. Jeśli się rozjadą — w którymś
języku brakuje przekładów.
